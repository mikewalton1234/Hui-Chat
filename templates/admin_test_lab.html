Browser release gate

Run these checks after the automated suite passes. They catch real Chrome/Firefox media and reconnect behavior that static tests cannot fully prove.

1. Automated gate

Run readiness, full suite, live user flow, and P2P browser check. All should pass before moving to manual browser media checks.

2. Chrome webcam-only

Join a room, click Webcam only, confirm the browser asks for camera only, Voice stays inactive, refresh/reconnect, and voice does not auto-restore.

3. Firefox webcam-only

Repeat the webcam-only test in Firefox because media permissions and device labels can behave differently from Chrome.

4. Voice combinations

Test Voice only, Webcam only, Voice + Webcam, then turn Voice off while Webcam remains on. The microphone should stop and the camera should stay active.

5. Reconnect recovery

Restart the server or refresh during media use. The reconnect banner should clear, normal room join should recover, and voice should restore only when Voice was explicitly enabled.

6. After-test cleanup

Confirm no zz_test_* users stay online, normal users can join rooms, and the browser does not remain stuck on reconnecting after Test Lab finishes.
